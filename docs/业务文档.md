# EduSocratic 业务需求文档（PRD）

**版本：v1.0**
**日期：2026-07-24**
**状态：定稿**

---

## 一、产品定位

### 1.1 一句话描述

> 基于飞书群机器人的少儿思辨课 AI 评估系统：家长拍照/录音提交学生作品，AI 自动评分，老师审核确认，家长收到可视化成长反馈。

### 1.2 解决什么问题

| 现状痛点 | 系统解决方案 |
|----------|-------------|
| 老师每节课要逐个点评 7-10 个学生，重复劳动 | AI 自动生成结构化评估，老师只需确认/微调 |
| 家长不知道孩子思辨能力具体怎么样 | 每次课后收到可视化评估卡片 |
| 家长看不到孩子的进步趋势 | 周报/月报展示成长曲线 |
| 评估标准不统一，全凭老师主观 | 六维量化评估 + 年级锚定标准 |

### 1.3 不解决什么问题

| 不做 | 原因 |
|------|------|
| 不做课堂实时录音转写 | 多人混杂，ASR 无法区分说话人，技术不可行 |
| 不做学生直接操作 | 6-13 岁儿童不具备操作能力 |
| 不做自动排课/教务管理 | 不是本产品范围 |
| 不做多机构 SaaS | 比赛项目，单机构验证即可 |

## 二、角色定义

| 角色 | 是谁 | 核心诉求 | 与系统的交互方式 | 频率 |
|------|------|---------|----------------|------|
| **学生** | 1-7年级，6-13岁 | 被看见、被鼓励 | 不直接操作系统 | — |
| **家长** | 学生父母 | 知道孩子表现、看到进步 | 在飞书群发图片/语音，收私信通知 | 每节课后 |
| **授课老师** | 圆桌星球教师 | 快速了解学生表现，减少重复劳动 | 在飞书群 @机器人、点卡片按钮 | 每节课后 |
| **教研负责人** | 课程质量管理者 | 评估标准是否合理、AI 是否准确 | 查看统计数据（Phase 3） | 每周/月 |

### 关键认知

- **学生不是用户**，是"被评估对象"。学生永远不会打开这个系统。
- **老师是第一用户**。系统必须让老师觉得"省事了"。
- **家长是第二用户**。家长要的是"看得懂的成长反馈"，不是专业分数。
- **家长的操作必须极简**：拍照 → 发群 → 完事。不能要求家长填表单。

## 三、数据输入（详细）

### 3.1 输入方式总览

| 方式 | 操作者 | 场景 | 技术实现 | 优先级 |
|------|--------|------|---------|--------|
| **A. 手写作文拍照** | 家长 | 课后，学生写完思辨短文 | GPT-4o Vision OCR | Phase 2（核心） |
| **B. 语音消息** | 家长 | 课后，学生口述一段观点 | Whisper API ASR | Phase 2（补充） |
| **C. 老师手动输入** | 老师 | 课堂中/课后记录关键发言 | 直接解析文字 | Phase 1（MVP） |

### 3.2 方式 A：手写作文拍照（详细流程）

**前置条件：**
- 学生已完成一篇 200-500 字的思辨短文（课后作业）
- 家长有智能手机，能拍照

**操作步骤：**

| 步骤 | 谁做 | 具体操作 | 系统响应 |
|------|------|---------|---------|
| A1 | 家长 | 拍学生手写作文（1-3张照片） | — |
| A2 | 家长 | 在飞书班级群发送图片 | — |
| A3 | 家长 | @机器人 + 附带信息（可选） | 机器人收到消息事件 |
| A4 | 系统 | 判断元信息是否完整 | 完整→进入预处理；不完整→追问 |
| A5 | 系统 | 下载图片（飞书 API） | — |
| A6 | 系统 | GPT-4o Vision 识别手写字 | 得到文本 + 置信度 |
| A7 | 系统 | 判断识别质量 | 高→直接评估；低→进入确认流程 |

**家长发送的消息格式（三种都支持）：**

```
格式1（完整）：
@EduSocratic 评估 学生:小明 年级:3 主题:应不应该给动物穿衣服
[图片]

格式2（部分）：
@EduSocratic 小明 年级3
[图片]
（主题缺失 → 机器人追问）

格式3（最简）：
@EduSocratic
[图片]
（全部缺失 → 机器人追问）
```

**机器人追问（元信息不完整时）：**

```
收到图片！请补充以下信息（直接回复即可）：

学生姓名：？
年级：？（1-7）
讨论主题：？

回复示例：小明 年级3 主题:应不应该给动物穿衣服
```

**家长回复后，系统解析并继续。**

### 3.3 方式 B：语音消息（详细流程）

**前置条件：**
- 学生对着手机说一段话（30秒-2分钟）
- 家长帮录并发到群里

**操作步骤：**

| 步骤 | 谁做 | 具体操作 | 系统响应 |
|------|------|---------|---------|
| B1 | 家长 | 帮学生录一段语音 | — |
| B2 | 家长 | 在飞书群发送语音消息 | — |
| B3 | 家长 | @机器人 + 附带信息 | 机器人收到消息事件 |
| B4 | 系统 | 下载音频文件（飞书 API） | — |
| B5 | 系统 | Whisper API 转文字 | 得到文本 |
| B6 | 系统 | 文本清洗（去填充词） | — |
| B7 | 系统 | 判断质量 | 有效→评估；无效→提示重录 |

**语音要求：**
- 时长：10秒 - 3分钟
- 语言：普通话
- 内容：学生自己的观点表达（不是朗读课文）

### 3.4 方式 C：老师手动输入（详细流程）

**操作步骤：**

| 步骤 | 谁做 | 具体操作 | 系统响应 |
|------|------|---------|---------|
| C1 | 老师 | 在飞书群 @机器人 | — |
| C2 | 老师 | 输入评估请求（含学生发言文字） | 机器人收到消息 |
| C3 | 系统 | 解析指令，提取元信息 | — |
| C4 | 系统 | 直接进入评估流程 | 跳过预处理 |

**老师输入格式：**

```
@EduSocratic 评估
学生:小明
年级:3
主题:应不应该给动物穿衣服
发言:我觉得应该给动物穿衣服，因为动物会冷。但是鱼不需要，鱼在水里不会冷。
```

**简化格式（也支持）：**

```
@EduSocratic 小明 3年级 动物穿衣服
"我觉得应该给动物穿衣服，因为动物会冷。但是鱼不需要。"
```

### 3.5 输入校验规则

| 校验项 | 规则 | 不通过时的响应 |
|--------|------|---------------|
| 年级范围 | 1-7 的整数 | "请输入 1-7 年级哦～" |
| 学生姓名 | 非空，≤20字 | "请告诉我学生姓名" |
| 主题 | 非空，≤100字 | "请告诉我讨论主题" |
| 文本长度（评估用） | ≥ 10 字 | "内容太少了，至少写/说2-3句话才能评估" |
| 文本长度（上限） | ≤ 2000 字 | 截断前 2000 字，不拒绝 |
| 图片数量 | 1-5 张 | >5张："最多发5张图片哦" |
| 语音时长 | 10秒 - 3分钟 | <10秒："语音太短啦"；>3分钟："请控制在3分钟内" |
| 重复提交 | 同一学生 60 秒内重复 | "刚刚已经评估过啦～结果在上面👆" |

## 四、预处理（图片/语音 → 文字）

### 4.1 图片 OCR 流程

```
输入：1-5 张手写作文图片（飞书 image_key）
输出：识别文本 + 置信度 + 不确定字符数

步骤：
1. 通过飞书 API 下载图片到内存
   GET /im/v1/messages/{message_id}/resources/{file_key}?type=image

2. 调用 GPT-4o Vision
   - 每张图片单独识别
   - 多张图片按顺序拼接

3. 文本清洗
   - 去除识别出的页眉页脚（页码、日期等）
   - 保留段落结构
   - 统计 [?] 标记数量

4. 质量判断
   - confidence > 0.8 且 [?] ≤ 3 → 高质量，直接评估
   - confidence 0.5-0.8 或 [?] 4-10 → 中质量，进入确认流程
   - confidence < 0.5 或 [?] > 10 → 低质量，请求重拍
```

**GPT-4o Vision Prompt：**

```
你是一个专业的手写文字识别助手。请识别这张手写作文图片中的所有文字。

要求：
1. 按原文书写顺序逐字识别
2. 保留段落结构和换行
3. 不确定的字用 [?] 标记
4. 不要添加任何原文没有的内容
5. 不要纠正错别字（原文怎么写就怎么识别）
6. 如果有涂改，识别最终版本（涂掉的不算）
7. 只输出识别的文字，不要输出任何解释

输出格式：纯文本，不要 JSON，不要 markdown。
```

### 4.2 语音 ASR 流程

```
输入：一段音频文件（飞书 file_key，格式 opus/ogg）
输出：转写文本 + 时长

步骤：
1. 通过飞书 API 下载音频
   GET /im/v1/messages/{message_id}/resources/{file_key}?type=file

2. 格式转换（如需要）
   飞书语音格式：opus → 转为 wav/mp3（Whisper 支持）

3. 调用 Whisper API
   model: "whisper-1"
   language: "zh"
   prompt: "这是一段小学生的思辨课发言，内容是关于某个话题的观点表达"

4. 文本清洗
   - 去除纯填充词（连续的"嗯""啊""那个""就是"）
   - 保留有内容的口语表达（"我觉得""因为""所以"）
   - 去除 ASR 产生的重复片段

5. 质量判断
   - 文本 ≥ 10 字 → 有效
   - 文本 < 10 字 → "没有听清楚，能再说一遍吗？"
```

### 4.3 预处理失败处理

| 失败情况 | 判断条件 | 系统行为 | 用户看到什么 |
|----------|---------|---------|-------------|
| 图片无法下载 | 飞书 API 返回错误 | 重试 2 次 → 报错 | "图片下载失败，请重新发送📸" |
| 图片不是作文 | Vision 返回"非文字内容" | 不评估 | "这好像不是作文图片哦～请发手写作文的照片" |
| 图片太模糊 | confidence < 0.5 | 不评估 | "图片不太清晰📸 能重新拍一张吗？光线好一点、正对着拍～" |
| 识别不确定多 | [?] > 10 个 | 进入确认流程 | 发确认卡片给老师 |
| 音频无法下载 | 飞书 API 返回错误 | 重试 2 次 → 报错 | "语音下载失败，请重新发送🎤" |
| 音频太短 | duration < 10s | 不评估 | "语音太短啦，至少说10秒钟哦🎤" |
| ASR 结果为空 | 返回空字符串 | 不评估 | "没有听清楚🎤 能找个安静的地方再说一遍吗？" |
| ASR 结果太短 | 文本 < 10 字 | 不评估 | "说的内容太少了，能多说几句吗？" |

## 五、确认环节（识别结果确认）

### 5.1 触发条件

| 条件 | 说明 |
|------|------|
| OCR 置信度 0.5-0.8 | 识别可能不准 |
| OCR [?] 标记 4-10 个 | 有不确定字符 |
| ASR 结果看起来不通顺 | 简单规则判断（如连续重复、乱码） |
| 老师手动要求确认 | 老师回复"让我看看识别结果" |

**不触发确认的情况：**
- OCR 置信度 > 0.8 且 [?] ≤ 3 → 直接评估
- 老师手动输入文字 → 不需要确认（已经是文字了）

### 5.2 确认流程

```
系统 → 老师（飞书群卡片）：

┌─────────────────────────────────────────────────────────┐
│  📝 识别结果确认 - 小明                                  │
│                                                         │
│  来源：手写作文拍照（2张图片）                           │
│  识别置信度：72%                                        │
│                                                         │
│  识别文字：                                             │
│  ┌─────────────────────────────────────────────────┐   │
│  │ 我觉得应该给动物穿衣服，因为动物会冷。           │   │
│  │ 但是鱼不需要，鱼在水里不会冷。[?]他动物         │   │
│  │ 可能也不需要，比如蛇，蛇有[?]。                 │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  ⚠️ 有 2 处不确定（[?]标记）                           │
│                                                         │
│  [✅ 确认无误，直接评估]                                │
│  [✏️ 我来修正文字]                                     │
│  [🗑️ 取消，不评估了]                                   │
└─────────────────────────────────────────────────────────┘
```

### 5.3 确认操作

| 操作 | 后续流程 |
|------|---------|
| ✅ 确认无误 | [?] 处取最可能字 → 直接进入评估 |
| ✏️ 修正文字 | 机器人回复"请发送修正后的完整文字" → 老师发文字 → 用修正文字评估 |
| 🗑️ 取消 | 不评估，记录取消原因 |

### 5.4 超时处理

| 超时时间 | 行为 |
|----------|------|
| 2 小时未操作 | 自动视为"确认无误"，用识别结果直接评估 |
| 评估卡片上标注 | "⚠️ 识别结果未经老师确认，自动评估" |

## 六、AI 评估（核心管道）

### 6.1 评估输入

| 字段 | 来源 | 必填 | 说明 |
|------|------|------|------|
| text | 预处理输出 / 老师输入 | ✅ | 学生发言/作文文字 |
| grade | 元信息 | ✅ | 年级 1-7 |
| topic | 元信息 | ✅ | 讨论主题 |
| student_id | 元信息 | ✅ | 学生标识 |
| class_id | 飞书群 chat_id | ✅ | 班级标识 |
| source_type | 系统判断 | ✅ | handwriting / voice / text |
| teacher_context | 老师打回时补充 | ❌ | 课堂上下文补充 |
| retry_count | 系统记录 | ✅ | 当前是第几次评估（0/1/2） |

### 6.2 评估输出

| 字段 | 类型 | 说明 |
|------|------|------|
| scores | dict | 六维分数，每维 1-5 整数 |
| weighted_score | float | 年级加权总分，1.00-5.00 |
| reasoning | string | 评分理由（给老师看） |
| highlights | string | 亮点（给家长看） |
| suggestions | string | 建议（给家长看） |
| overall_comment | string | 总评（给家长看） |
| needs_review | bool | 是否需要人工复核 |

### 6.3 六维评估体系

| 维度 | 评估什么 | 1分表现 | 5分表现 |
|------|---------|---------|---------|
| 观点清晰度 | 有没有明确立场 | 没有观点/跑题 | 观点明确、表述清晰 |
| 逻辑连贯性 | 论证有没有逻辑链 | 前后矛盾/跳跃 | 因果链完整、层次分明 |
| 证据支撑 | 有没有用事实/例子 | 纯主观"我觉得" | 有具体事例/数据支撑 |
| 多角度思考 | 有没有考虑反方 | 只有一面 | 主动考虑对立面 |
| 回应与质疑 | 有没有回应他人观点 | 自说自话 | 能针对性反驳/追问 |
| 表达完整性 | 表达是否完整通顺 | 词不达意/残缺 | 表达流畅、结构完整 |

### 6.4 年级差异化

#### 年级权重

| 维度 | 1-2年级 | 3-4年级 | 5-7年级 |
|------|---------|---------|---------|
| 观点清晰度 | 0.30 | 0.25 | 0.20 |
| 逻辑连贯性 | **0.00** | 0.15 | 0.20 |
| 证据支撑 | 0.25 | 0.20 | 0.20 |
| 多角度思考 | **0.00** | 0.15 | 0.20 |
| 回应与质疑 | 0.20 | 0.15 | 0.10 |
| 表达完整性 | 0.25 | 0.10 | 0.10 |

**1-2年级不评估"逻辑连贯性"和"多角度思考"**（认知发展阶段未达到）。

#### 年级锚定（Prompt 中注入）

```
1-2年级锚定：
- 能用一句话说出"我觉得..."就是3分
- 能说出"因为..."就是4分
- 不要求逻辑严密，不要求多角度

3-4年级锚定：
- 有观点+一个理由=3分
- 有观点+理由+例子=4分
- 能提到"但是反方可能觉得..."=5分

5-7年级锚定：
- 有观点+理由+例子=3分
- 有完整论证链+考虑反方=4分
- 能主动质疑+提出新角度=5分
```

### 6.6 内容预检规则

| 检查项 | 条件 | 结果 |
|--------|------|------|
| 太短 | len(text) < 10 | 拒绝："内容太少了" |
| 无中文 | 不含任何中文字符 | 拒绝："未检测到中文内容" |
| 无意义 | 去重后字符 < 5 且总长 > 20 | 拒绝："内容似乎无意义" |
| 太长 | len(text) > 2000 | 截断前 2000 字，继续评估 |
| 正常 | 以上都不触发 | 继续 |

### 6.7 LLM 调用规格

| 参数 | 值 | 说明 |
|------|-----|------|
| model | gpt-4o-mini | 成本优先 |
| temperature | 0 | 确定性输出 |
| max_tokens | 2000 | 足够输出完整 JSON |
| timeout | 30s | 超时进入兜底 |
| 输出格式 | JSON | Prompt 中严格要求 |

### 6.8 输出校验（三段式）

| 阶段 | 校验内容 | 失败处理 |
|------|---------|---------|
| 第一段：格式 | 是否为合法 JSON | 修复重试 |
| 第二段：结构 | 是否包含所有必需字段 | 修复重试 |
| 第三段：值域 | 分数是否 1-5 整数 | 修复重试 |

**修复重试 Prompt：**

```
你上次的输出格式有误，无法解析。

错误信息：{error_message}
你的原始输出：{raw_output[:500]}

请严格按照以下 JSON 格式重新输出，不要输出任何其他内容：
{schema_example}
```

### 6.9 兜底策略

| 触发条件 | 兜底行为 |
|----------|---------|
| LLM 超时（>30s） | 六维全 3 分，needs_review=true |
| LLM 返回非 JSON 且修复 2 次仍失败 | 六维全 3 分，needs_review=true |
| LLM API 报错（429/500） | 六维全 3 分，needs_review=true |

**兜底时的用户提示：**
```
⚠️ 评估暂时不可用，已标记为待人工复核。
系统会自动重试，或老师可以稍后手动触发重新评估。
```

## 十、反馈闭环

### 10.1 反馈数据收集

每次老师操作都记录到 teacher_feedbacks 表：

| 操作 | 记录内容 |
|------|---------|
| confirm | assessment_id, action=confirm |
| modify | assessment_id, action=modify, modified_scores, reason |
| disagree | assessment_id, action=disagree, comment |
| retry | assessment_id, action=retry, teacher_context, new_scores |

### 10.2 偏差检测（Phase 3）

**触发条件：** 累积 20 条以上反馈

**检测逻辑：**

```
对每个维度：
  修改率 = modify次数 / 总评估次数
  方向 = 老师改高次数 vs 老师改低次数

  如果 修改率 > 30% 且 方向一致（>70%同方向）：
    → 判定该维度存在系统性偏差
    → 生成 Prompt 修改建议
```

### 10.3 Prompt 优化流程

```
偏差检测触发
    │
    ▼
系统生成修改建议（LLM 辅助）
    │
    ▼
发送给教研负责人（飞书卡片）
    │
    ▼
教研负责人审批：
    ├── 批准 → 新版本 Prompt 上线（v1.0 → v1.1）
    ├── 修改后批准 → 教研负责人调整后上线
    └── 拒绝 → 记录拒绝原因，不修改
```

**规则：**
- Prompt 修改必须人工审批，不能自动应用
- 每次修改生成新版本号
- 新 Prompt 不追溯历史数据
- 可随时回滚到上一版本

## 十一、飞书交互设计

### 11.1 系统架构

```
┌──────────┐         ┌──────────────┐         ┌──────────────┐
│  飞书群   │         │  飞书服务器   │         │  你的后端     │
│          │         │              │         │  (FastAPI)   │
│ 老师/家长 │──消息──→│  事件推送     │──HTTP──→│  /webhook    │
│          │         │              │         │              │
│          │←─卡片──│  消息API     │←─HTTP──│  调飞书API   │
│          │         │              │         │              │
│          │──按钮──→│  回调推送     │──HTTP──→│  /callback   │
└──────────┘         └──────────────┘         └──────────────┘
```

**后端只需暴露 2 个 HTTP 端点：**
1. `POST /webhook` — 接收飞书推送的消息事件（图片/语音/文字）
2. `POST /callback` — 接收卡片按钮点击回调

### 11.2 消息类型处理

| 飞书消息类型 | 触发条件 | 系统行为 |
|-------------|---------|---------|
| text + @机器人 | 老师/家长 @机器人发文字 | 解析指令（评估/查询/帮助/生成周报） |
| image + @机器人 | 家长发图片 @机器人 | 进入 OCR 预处理流程 |
| audio + @机器人 | 家长发语音 @机器人 | 进入 ASR 预处理流程 |
| text（无@） | 在追问上下文中回复 | 作为补充信息处理 |
| card.action.trigger | 点击卡片按钮 | 处理确认/打回/修改/取消 |

### 11.3 机器人指令列表

| 指令 | 格式 | 说明 |
|------|------|------|
| 评估 | `@机器人 评估 学生:X 年级:N 主题:Y [文字/图片/语音]` | 触发评估 |

| # | 场景 | 触发条件 | 系统行为 | 用户看到什么 |
|---|------|---------|---------|-------------|

| E2 | 不是作文 | Vision 判断非文字 | 不评估 | "这好像不是作文图片哦～" |

| E5 | 语音噪音大 | ASR 返回空 | 不评估 | "没有听清楚🎤 能再说一遍吗？" |

## 十七、术语表

| 术语 | 含义 |
|------|------|
| 六维评估 | 观点清晰度、逻辑连贯性、证据支撑、多角度思考、回应与质疑、表达完整性 |
| 年级锚定 | 同一表现在不同年级给不同分数的标准 |

| 兜底 | LLM 失败时返回中性 3 分，标记待人工复核 |
| 熔断 | 配额用完后拒绝新请求，防止成本失控 |
| 确认率 | 老师直接确认（不修改）的比例，衡量 AI 准确率 |
